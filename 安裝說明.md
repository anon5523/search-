# 巡檢通報系統 — 安裝與部署說明

這是一套**離線優先的 PWA**（漸進式網頁應用）。手機開網址即用、可加到主畫面像 App，
沒網路也能拍照建檔，回到收訊自動補傳到你的 **Google 試算表**，照片影片存 **Google Drive**。
Android / iOS 皆可使用，**不需上架 App Store**。

---

## 系統有兩個網址（很重要）

| 網址 | 用途 | 誰來提供 |
|---|---|---|
| **① App 網址** | 承辦人手機開啟的網頁（放 `index.html` 的地方） | 需要你「上架」到免費靜態空間，見下方步驟 A |
| **② Apps Script 網址** | 後端，負責把資料寫進 Google 試算表 | 你部署 Apps Script 後自動產生，貼進 App 設定，見步驟 B |

> 這兩個網址都必須由你的帳號建立——我無法替你產生線上網址，但下面照做 10 分鐘可完成。

---

## 步驟 A：把 App 放上網（擇一即可）

### 最簡單：Netlify Drop（免帳號、拖曳即可）
1. 開啟 https://app.netlify.com/drop
2. 把整個 `inspection-app` 資料夾**整包拖進去**
3. 幾秒後得到一個 `https://xxxx.netlify.app` 網址 → 這就是 **App 網址①**

### 替代方案：GitHub Pages（適合長期維護）
1. 建立一個 GitHub repo，把所有檔案上傳
2. Settings → Pages → Branch 選 `main` → 儲存
3. 取得 `https://你的帳號.github.io/repo名/` 網址

> ⚠️ 必須是 **HTTPS** 網址，離線功能與相機定位才會啟用。用本機 `file://` 直接開無法定位。

---

## 步驟 B：設定 Google 試算表後端

1. 到 https://sheets.google.com 新建一份試算表（名稱自取，例如「巡檢資料庫」）
2. 上方選單 **擴充功能 → Apps Script**
3. 刪掉預設內容，貼上 `Code.gs` 全部內容，存檔
4. 右上角 **部署 → 新增部署作業**
   - 類型選 **網頁應用程式**
   - 「執行身分」：**我自己**
   - 「具有存取權的使用者」：**任何人**
   - 按部署，首次會要求授權 → 允許
5. 複製產生的 **網頁應用程式網址**（結尾是 `/exec`）→ 這就是 **Apps Script 網址②**

> `Code.gs` 已內建你的試算表 ID（`1-7-VSme…CNLjbw`），資料會自動寫進那一份，
> 不論用「綁定試算表」或「獨立專案」貼上都可以。要換別份試算表時，改開頭的 `SHEET_ID` 即可。
>
> **頁籤規則：每個專案各自一個頁籤、以專案名稱命名；未選專案者進「未分類」頁籤。**
> 首次上傳時會自動建立對應頁籤與「巡檢照片」Drive 資料夾，無需手動建立。
> 改善紀錄（最新一次的單位／工法／狀態／照片）會併入該列摘要欄位；完整多次改善歷程保留在 App。

---

## 步驟 C：啟用 App

1. 手機開啟 **App 網址①**
2. 進入 **設定**：填寫「巡檢人員姓名」、貼上 **Apps Script 網址②**，儲存
3. 按「測試連線」出現「連線成功 ✓」即完成
4. **加到主畫面**：
   - Android Chrome：右上 ⋮ →「安裝應用程式 / 加到主畫面」
   - iPhone Safari：分享鈕 →「加入主畫面」

---

## 功能總覽

- **新增巡檢**：所屬專案、行政區、GPS 定位、破損類別（一般道路／防汛道路／護岸／其他，可複選）、
  破損狀況、其他描述、現場照片影片。照片自動壓縮、影片有大小上限提醒。
- **專案管理**：自建／編輯專案名稱與「事件期間」（日曆選起訖日），新增巡檢時即可歸類，清單可依專案篩選。
- **改善追蹤**：每筆紀錄可新增多次改善（改善單位、改善工法、改善後狀態、改善照片），狀態自動更新為「改善中／已改善」。
- **離線補傳**：沒網路時存在手機，畫面顯示「待上傳」，回到收訊自動上傳，「項次」由試算表統一編號回填。
- **自動排錯**：必填欄位紅框提示、GPS 失敗改手動填地標、座標精度顯示、影片超量擋下、項次防重複、重複上傳改為更新不重複建列。

---

## 自訂設定（改一個檔就好）

打開 `index.html`，找到最上方 `const CONFIG = {`：

- `AREAS`：改成你的縣市行政區清單（目前範例為新北市）
- `CATEGORIES`：破損類別選項
- `VIDEO_MAX_MB`：影片大小上限（預設 25MB）
- `IMG_MAX_EDGE` / `IMG_QUALITY`：照片壓縮品質

改完重新上傳到步驟 A 的空間即可。

---

## 注意事項

- 影片建議短片段；單檔過大上傳易失敗（Apps Script 單次約 50MB 上限），預設擋在 25MB。
- 清除手機瀏覽器資料前，請先確認清單內紀錄皆顯示「已上傳」。
- 多位承辦人各自用自己手機即可，項次由各專案頁籤內連號、不會打架。
- **專案改名請趁早**：若紀錄已上傳後才改專案名稱，試算表既有頁籤不會自動更名，之後該專案的新資料會寫進新名稱的頁籤。建議建立專案時就定好名稱。
- iOS 需用 **Safari** 安裝才支援加到主畫面；Android 用 **Chrome**。
