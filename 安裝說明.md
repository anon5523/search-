# 巡檢通報系統 — 安裝與部署說明（臺南市版）

離線優先的 **PWA**：手機開網址即用、可加到主畫面像 App，沒網路也能拍照標點建檔，
回到收訊自動補傳到 **Google 試算表**，照片影片存 **Google Drive**。Android / iOS 皆可，**免上架 App Store**。

> **承辦人完全不用做任何設定**：第一次開啟只要輸入姓名，就直接進入巡檢填報。
> 後端網址由「管理者」在程式內填一次即可（步驟 2）。

---

## 一次性建置（管理者，約 10 分鐘）

### 步驟 1：部署 Google 試算表後端，取得網址
1. 用要存資料的 Google 帳號，到 https://sheets.google.com 開一份試算表（或用你已建立的那份）
2. **擴充功能 → Apps Script**，刪掉預設內容，貼上 `Code.gs` 全部內容，存檔
3. **部署 → 新增部署作業**
   - 類型：**網頁應用程式**
   - 執行身分：**我自己**
   - 具有存取權的使用者：**任何人**
   - 部署 → 首次會要求授權 → 允許
4. 複製產生的網址（結尾 `/exec`）

> `Code.gs` 已內建你的試算表 ID（`1-7-VSme…CNLjbw`），資料會寫進那一份；要換別份改開頭 `SHEET_ID` 即可。
> **每個專案各自一個頁籤（以專案名命名）、未選專案進「未分類」頁籤**，首次上傳自動建立頁籤與「巡檢照片」資料夾。

### 步驟 2：把網址填進 App（關鍵，承辦人因此免設定）
打開 `index.html`，最上方 `const CONFIG = {` 裡：
```
APPS_SCRIPT_URL: "",   ← 把步驟 1 的 /exec 網址貼進這對引號中
```
例如：`APPS_SCRIPT_URL: "https://script.google.com/macros/s/AKfy.../exec",`

### 步驟 3：把整包資料夾上架（HTTPS）
- **最簡單**：開 https://app.netlify.com/drop ，把整個 `inspection-app` 資料夾**整包拖進去**，幾秒得到 `https://xxxx.netlify.app` 網址。
- 或 **GitHub Pages**：上傳整包 → Settings → Pages → 選 `main` → 取得網址。

> ⚠️ 必須是 **HTTPS**；用 `file://` 直接開無法定位與離線。整包要含 `assets/leaflet/`（地圖元件）才會正常。

### 步驟 4：把網址發給承辦人
傳上面的 App 網址給同仁即可。

---

## 承辦人使用（零設定）
1. 開啟 App 網址 → 第一次輸入**姓名** → 直接進入**巡檢填報**
2. **加到主畫面**：Android Chrome 右上 ⋮ →「安裝應用程式」；iPhone 用 **Safari** 分享鈕 →「加入主畫面」

---

## 功能總覽（底部五個分頁）

- **填報**：選專案 → 臺南市行政區 → **地圖標點**（點地圖／拖圖釘／或按鈕定位到目前位置）→ 破損類別（可複選；勾「其他破損」會跳出說明欄）→ 破損狀況、其他描述、現場照片影片。
- **查詢**：依狀態、行政區、專案篩選清單。
- **地圖**：所有破損點顯示於地圖，可**雙指縮放、拖曳**，依行政區聚焦，點圖釘看摘要並開啟詳情。
- **改善**：獨立填報頁，挑出「待改善／改善中」紀錄 → 填改善單位、工法、改善後狀態與改善照片。
- **設定**：巡檢人員姓名、進入專案管理、後端連線狀態。
- **專案管理**：建立／編輯專案名稱與**事件期間（日曆選起訖日）**，填報時即可歸類。

---

## 自訂（改 `index.html` 最上方 CONFIG）
- `AREAS`：行政區清單（已設為臺南市 37 區）
- `CATEGORIES` / `OTHER_CAT`：破損類別、哪一項要跳出說明欄
- `MAP_CENTER` / `MAP_ZOOM`：地圖預設中心與縮放（已設臺南）
- `VIDEO_MAX_MB`、`IMG_MAX_EDGE`、`IMG_QUALITY`：影片上限與照片壓縮
改完重新上架即可。

---

## 注意事項
- 地圖底圖需要網路；**離線時仍可拍照、GPS 標點、建檔**，回到收訊自動上傳。
- 影片建議短片段，預設擋在 25MB（Apps Script 單次約 50MB 上限）。
- 清除手機瀏覽器資料前，請先確認紀錄皆顯示「已上傳」。
- **專案改名請趁早**：紀錄上傳後才改專案名，試算表既有頁籤不會自動更名。
- 多位承辦人各用自己手機即可，項次於各專案頁籤內連號、不衝突。
